Multi-Resource Packing for Cluster Schedulers

Summary:

This paper introduces Tetris, a multi-resource cluster scheduler that packs
tasks to machines based on their requirements of all resource types. Previous
scheduling policies may result in waste due to resource (e.g. CPU, memory)
fragmentation and over-allocation. Also, fairness concern further degrades
the performance. This paper aims to strike a balance between efficiency,
job completion time and fairness.

To achieve all goals,Tetris adapts heuristics for the multi-dimensional bin
packing problem to the context of cluster schedulers. In addition, Tetris
improves average job completion time by prioritizing jobs that have less re-
maining work. Trace-driven simulations and deployment of their Apache YARN
prototype on a 250 node cluster show gains of over 30% in makespan and job
completion time while achieving nearly perfect fairness.

PISCES A Programmable, Protocol-Independent Software Switch

Summary:

This paper presents PISCES, a software switch derived from Open vSwitch
(OVS), a hard-wired hypervisor switch, whose behavior is customized using
P4. PISCES is not hard-wired to specific protocols; this independence makes
it easy to add new features. Authors show how the compiler can analyze the
high-level specification to optimize forwarding performance. The evaluation
shows that PISCES performs comparably to OVS and that PISCES programs are
about 40 times shorter than equivalent changes to OVS source code.

This prototype demonstrates the feasibility of a protocol-independent soft-
ware switch using P4 as the programming language and OVS as the target swi-
tch. Moreover, techniques for software switch protocol independence and for
compiling a domain-specific packet-processing language to an efficient low-
level implementation should generalize to other languages and targets.

TensorFlow

Summary:

This paper presents a system, known as “TensorFlow”, for expressing parallel
and distributed machine learning algorithms. TensorFlow provides flexible i-
nterfaces for quickly experimenting new machine learning models. It allows
clients to express various kinds of parallelism through replication and par-
allel execution of a core model dataflow graph. The computation model is de-
scribed by a directed graph, with nodes representing input and edges storing
tensors. The main componets in TensorFlow are client, which uses the Session
interfaces to communicate with the master, and one or more worker processes,
with each workder process responsible for arbitrating access to one or more
computational devices. It has both local and distributed execution workflow.

In summary, this paper describes a system facilitating single machine or di-
stributed implementation of tensor-related algorithms. The source code of t-
tensorflow is available on github.

Packet Transaction High-Level Programming for Line-Rate Switches

Summary:

This paper shows how to program data-plane algorithms in a high-level lang-
uage and compile those programs into low-level microcode that can run on e-
merging programmable line-rate switching chips. The key challenge is that
many data-plane algorithms create and modify algorithmic state. Authors in-
troduce the notion of a packet transaction: a sequential packet-processing
code block that is atomic and isolated from other such code blocks. With t-
his abstraction, switch is able to process packets in line-rate.

In summary, this paper defines a block of imperative code as a packet tran-
saction, which is then composed of several atoms (instructions set executed
exclusively). Finally, it gives a detailed description on the compiler, wh-
ich compiles packet transaction to atoms.

RDMA over Commodity Ethernet at Scale

Summary:

With the requirement of today’s low delay network, researchers
in Microsoft have deployed RDMA to address TCP’s latency and CPU
overhead problems. The technology applied is RoCEv2, known as
RDMA over Converged Ethernet.

Unlike TCP, RDMA needs a lossless network. However, this is not
achieved with zero cost. The authors describe some problems en-
countered during their large scale deployment of RDMA in the DCN
of Microsoft. The most important are: 1) RDMA transport livelock;
2) PFC deadlock; 3) PFC pause frame storm; 4) Slow-receiver sym-
ptom. The paper also highlights an open problem: How to achieve
low latency and high throughput at the same time for RDMA in DCN.
The fact is that A lossless network does not guarantee low latency.

Ernest Efficient Performance Prediction for Large-Scale Advanced Analytics

Summary:

The rapid adoption of advanced analytics workloads makes it important to co-
nsider how these applications can be deployed in a cost and resource effici-
ent fashion.

This work predicts performance of applications under various resource confi-
gurations provide end users with the optimal configuration. The insight is
that a number of jobs in today’s distributed computing frameworks have pred-
ictable structure in terms of computation and communication. Authors build
performance models based on the behavior of the job on small samples of data
and then predict its performance on larger datasets and cluster sizes with
low overhead.

FUSO Leveraging Multi-path Diversity for Transport Loss Recovery in Data Centers

Summary:

In modern DCN, packet loss is inavoidable, mainly due to cong-
estion loss and hardware failures. Such loss will hurt the
tail latency of network flows. conventional TCP applies fast
recovery and timeout (RTO) to mitigate the loss. However,
in DCNs, the RTO is costly, which lead to many efforts in
avoiding RTO. Previous works more or less introduce large
redundency.

This work leverages multi-path diversity in DCN for transport
loss recovery. In FUSO, when a multi-path transport sender
suspects loss on one sub-flow, recovery packets are immediately
sent over another sub-flow that is not or less lossy and has
spare congestion window slots.

Rehearsal of CODA and Karuna

Summary:

Developed from coflow series, Varys - Coflow with complete
information, Aalo - coflow with incomplete information,
CODA - identify coflows without modifying applications
CODA is both application-transparent and error-tolerant.

Previous scheduling policies (e.g. pFabric, PASE, PIAS, PDQ)
cannot satisfy mix-flows. Hence, Karuna is designed to deal
with the case where deadline-aware and non-deadline flows
are mixed. It challenges to accomodate deadline flows (
deadline sensitive) and non-deadline flows (FCT sensitive).
The solution is threefold: prioritize deadlin flows,
use minimal bandwidth to complete before deadline and finally
reduce FCT for non-deadline flows with remaining bandwidth.

NUMFabric Fast and Flexible Bandwidth Allocation in Datacenters

Summary:

This work proposes NUMFabric, a novel transport design that provides
flexible and fast bandwidth allocation control. NUMFabric enables
operators to flexibly optimize resource allocation in order to meet
service level objectives. By decoupling network utilization from resource
allocation via explicit packet scheduling in the network, NUMFabric
shows how sophisticated packet scheduling can help implement diverse
policies at datacenter scales.

However, it is still an open question as to how higher
layer system can take advantage of NUMFabric’s capabilities
to optimize application layer objectives and this is
the focus of our current work. We are extending NUMFabric
to more general definitions of flows such as coflows,
VM-level aggregates and tenant-level aggregates.
