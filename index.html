Flowtune Flowlet Control for Datacenter Networks

Summary:

This work propose a centralized bandwith allocation system called Flwotune
based on the granularity of flowlets. With flowlet control, allocations have
to change only when flowlets arrive or leave. The authors apply the Network
Utility Maximization (NUM) to make resource allocation decisions. A detailed
scaling method in multi-core environment is introduced.

Experiments show that Flowtune outperforms DCTCP, pFabric, sfqCoDel, and XCP
on tail packet delays in various settings, converging to optimal rates within
a few packets rather than over several RTTs.

Let It Flow Resilient Asymmetric Load Balancing with Flowlet Switching

Summary:

This work implement a simplified and stateless load balancing system named
LeetFlow on switches, leveraging the elastic property of flowlets. Flowlets
have the property: their size changes automatically based on traffic conditions
on their path. This property is proved to be efficient to build a balancing
scheme that is resilient to asymmetry.

Despite being much simpler, it performs significantly better than other traffic
oblivious schemes like WCMP and Presto in asymmetric scenarios, while achieving
average flow completions time within 10-20% of CONGA in testbed experiments.

Sunflow Efficient Optical Circuit Scheduling for Coflows

Summary:

This paper presents the first Optical Circuit Switch scheduling algorithm.
Compared with traditional synchronous circuit switching model, SunFlow
leverage an asynchronous model to maximize the makespan of Coflow, while
keeping the circut switch overhead low. Sunflow realizes non-preemptive
scheduling for intra-Coflow, not allowing subflows to preempt each other.
For inter-Coflow scheduling, Sunflow implements flexible preemption policy
to mimic shortest-Coflow-first. At the intra-Coflow level, the algorithm
is proved a factor 2 performance guarantee for the optimal cricuit switch
scheduler.

Using RDMA Efficiently for Key-Value Services

Summary:

This paper describes the design and implementation of HERD, a key-value system
designed to make the best use of an RDMA network. Unlike prior RDMA-based key-
value systems, HERD focuses its design on reducing network round trips while
using efficient RDMA primitives; the result is substantially lower latency, and
throughput that saturates modern, commodity RDMA hardware.

BBR Congestion-based congestion control

Summary:

Internet congestion problems result from a design choice made when TCP conges-
tion control was created in the 1980s-�—interpreting packet loss as “congest-
ion.” This equivalence was true at the time but was because of technology lim-
itations, not first principles. As NICs (network interface controllers) evolved
from Mbps to Gbps and memory chips from KB to GB, the relationship between pac-
ket loss and congestion became more tenuous.

Today TCP’s loss-based congestion control-�—even with the current best of breed
, CUBIC�–is the primary cause of these problems. When bottleneck buffers are
large, loss-based congestion control keeps them full, causing bufferbloat. When
bottleneck buffers are small, loss-based congestion control misinterprets loss
as a signal of congestion, leading to low throughput. Fixing these problems req-
uires an alternative to loss-based congestion control. Finding this alternative
requires an understanding of where and how network congestion originates.

Altruistic scheduling in multi-resource clusters

Summary:

The main thing of the paper is evolving the “altruistic” design in the algorithms
, i.e. a job can be delayed to make room for other jobs so as to make room for
other jobs to access the computing resources. After delay stage other jobs greed-
ily fills the empty space for the jobs. This has even more significant impact as
jobs has more complicated DAGs. Based on the observation from another OSDI paper
“Graphene”, discussing the distribution of the complexity of DAGs which made the
designs in this paper meaningful, and the work of “Tetris” (also from the same
author) which implements a task assigning framework that tried to minimize the
makespan of the cluster, an altruistic scheduling scheme was made to approximate
the best known performance metric published in conferences and in practice: DRF
for fairness, Tetris for makespan and Shortest-Job First for average job comple-
tion time(JCT).

HyPer4 Using P4 to Virtualize the Programmable Data Plane

Summary:

In this paper, researchers propose HyPer4 as a portable virtualization solution.
HyPer4 provides a general purpose program, written in the P4 dataplane programm-
ing language, that may be dynamically configured to adopt behavior that is func-
tionally equivalent to other P4 programs. HyPer4 permits users to modify the set
of programs, as well as the virtual network connecting them, at runtime, without
disrupting currently active programs.

In many cases network operators might desire more than one context for a given
network device even if this device has only one physical data plane. One answer
is virtualization. This technique permits the illusion of multiple programmable
data planes and greater flexibility as to how they are used.

Trumpet Timely and Precise Triggers in Data Centers

Summary:

As data centers evolve to larger scales, higher speed, and higher link util-
ization, monitoring systems must detect events (such as transient congestion
or server load imbalance) more precisely (inspecting every packet), at fine
time-scales (on the order of milliseconds) and in a programmable fashion (so
that the set of events can be dynamically determined).

In this paper, the researchers present the architecture and implementation of
Trumpet: an event monitoring system in which users define network-wide events
, a centralized controller installs triggers at end-hosts where triggers test
for local conditions, and the controller aggregates these signals and tests
for the presence of specified network-wide events.

Taking the Blame Game out of Data Centers Operations with NetPoirot

Summary:

In this paper, the authors pose the question: how much can you infer about
a failure in the data center using TCP statistics collected at one of the
endpoints. Using an agent that captures TCP statistics NetPoirot devises a
classification algorithm that identifies the root cause of failure using this
information at a single endpoint. Based on the inferred results by applying
the model to TCP metrics, data center network operators may be able to predict
potential failures.

Network Requirements for Resource Disaggregation

Summary:

Recent industry trends suggest a paradigm shift to a disaggregated data cen-
ter (DDC) architecture containing a pool of resource, each built as a stand-
alone resource blade and interconnected using a network fabric. To support
numerous applications on such an architecture, it becomes critical that the
network fabric provide low latency communication. Hence, in this paper, aut-
hors implement a bunch of evalutions to measure the minimum (bandwidth and
latency) requirements that the network in disaggregated datacenters must pr-
ovide. By adding memory access delay into the applications, the emulations
measure the difference between conventional distributed computing frameworks
and this new DDC architecture.

In all, this paper provides multiple workloads-driven emulations to show the
potential network requirements in a disaggregated datacenter.
